# PRD: 공학용 계산기 (Scientific Calculator)
*가제 — 확정 필요.*

## 0. 문서 목적
이 PRD는 학생용 웹 계산기의 MVP 범위와 요구사항을 정의한다. 대상 독자는 PM 본인과 이후 UX·아키텍처·구현 워크플로우 담당자다. 용어는 §3 Glossary로 고정하고, 기능은 그룹으로 묶어 그 아래 FR을 중첩한다. 추정한 부분은 본문에 `[ASSUMPTION]`으로 표시하고 §9에 모았다. MVP는 "최대한 간결" 방향에 따라 기본 사칙연산으로 좁히며, 공학 함수는 향후 확장으로 둔다.

## 1. Vision
학생이 브라우저에서 즉시 열어 군더더기 없이 계산할 수 있는 웹 계산기다. 설치·로그인·설정 없이 화면을 열면 바로 숫자를 누르고 결과를 얻는다. 복잡한 공학용 기능을 한꺼번에 욱여넣기보다, 기본 사칙연산을 빠르고 정확하고 오류 없이 처리하는 것을 1차 목표로 한다.

간결함이 핵심 가치다. 버튼은 적게, 화면은 깨끗하게, 동작은 예측 가능하게. React로 구현하여 키보드와 마우스 모두에서 자연스럽게 동작하고, 학생이 수업·과제 중 빠르게 쓰고 닫는 도구가 된다. 공학 함수(삼각·로그·지수 등)는 동일한 간결함을 유지한 채 이후 확장 단계에서 더한다.

## 2. Target User

### 2.1 Jobs To Be Done
- 과제·수업 중 **빠른 산수**가 필요할 때 새 탭에서 즉시 계산하고 싶다.
- 휴대폰 계산기나 OS 기본 계산기를 찾는 대신 **브라우저 안에서** 해결하고 싶다.
- 계산 결과를 **신뢰**하고 싶다 — 잘못된 입력에도 앱이 멈추거나 틀린 값을 주지 않기를 바란다.
- [ASSUMPTION] 나중에 **공학 함수**(sin/cos/log 등)가 필요해질 때 같은 도구를 계속 쓰고 싶다.

### 2.2 Non-Users (v1)
- 그래프·방정식 풀이·기호 연산(CAS)이 필요한 사용자 — v1 대상 아님.
- 프로그래머/진법 변환·비트 연산이 필요한 사용자 — v1 대상 아님.

### 2.3 Key User Journeys
- **UJ-1. 지민이 과제를 풀다 빠르게 곱셈을 확인한다.** 지민은 통계 과제를 하다 막혀 새 탭에서 계산기를 연다. 화면이 즉시 뜨고, 키보드로 `128*0.25`를 입력한 뒤 Enter를 누르면 `32`가 크게 표시된다. 값을 확인하고 탭을 닫는다. **Edge case:** 실수로 `5/0`을 입력하면 앱이 멈추지 않고 "정의되지 않음" 같은 명확한 오류 표시를 보여주고, 다음 입력을 바로 받는다.

## 3. Glossary
- **계산기(Calculator)** — 본 제품. 단일 화면의 웹 애플리케이션.
- **수식(Expression)** — 사용자가 입력한 연산 문자열 (예: `12 + 3 × 4`).
- **결과(Result)** — 수식을 평가한 수치 출력.
- **디스플레이(Display)** — 현재 수식과 결과를 보여주는 화면 영역.
- **기본 사칙연산(Basic Arithmetic)** — 덧셈·뺄셈·곱셈·나눗셈, 그리고 이를 보조하는 소수점·부호변경·괄호.

## 4. Features

### 4.1 기본 계산 (Basic Calculation)
**Description:** 사용자는 디스플레이에 수식을 입력하고 결과를 얻는다. 버튼 클릭과 키보드 입력을 모두 지원하며, 디스플레이는 입력 중인 수식과 평가된 결과를 동시에 보여준다. 연산자 우선순위(곱·나눗셈이 덧·뺄셈보다 먼저)와 괄호를 따른다. 잘못된 입력은 앱을 중단시키지 않고 명확한 오류 메시지로 처리한다. UJ-1을 실현한다. 괄호는 MVP에 포함한다 — 우선순위 계산의 정확성을 위해 필요.

**Functional Requirements:**

#### FR-1: 숫자·연산자 입력
사용자는 버튼 또는 키보드로 숫자(0–9), 소수점, 사칙연산자(+ − × ÷), 괄호를 입력해 수식을 만들 수 있다. Realizes UJ-1.

**Consequences (testable):**
- 버튼 클릭과 대응 키보드 키(예: `*`→×, `/`→÷)가 동일하게 동작한다.
- 입력한 문자가 디스플레이의 현재 수식에 즉시 반영된다.
- 소수점은 한 숫자 안에 한 번만 입력된다(`1.2.3` 방지).

#### FR-2: 수식 평가
사용자는 `=` 버튼 또는 Enter 키로 현재 수식을 평가해 결과를 얻는다. Realizes UJ-1.

**Consequences (testable):**
- 연산자 우선순위와 괄호를 정확히 반영한다 (`2 + 3 × 4` = `14`).
- 결과는 입력 수식과 함께 디스플레이에 표시되고, 이어서 새 연산의 시작값으로 사용할 수 있다.
- 부동소수점 결과는 합리적 자리수로 표시된다. [ASSUMPTION] 표시 자리수 정책은 §8에서 확정.

#### FR-3: 오류 처리
잘못된 수식(0으로 나누기, 미완성 수식, 잘못된 괄호 짝)에 대해 계산기는 앱을 중단시키지 않고 명확한 오류 상태를 표시한다.

**Consequences (testable):**
- `5 ÷ 0` 입력 시 정의되지 않음을 나타내는 오류 메시지를 표시한다.
- 오류 상태에서 다음 입력을 받으면 정상 입력 모드로 복귀한다.

#### FR-4: 초기화/삭제
사용자는 전체 초기화(C/AC)와 마지막 입력 삭제(백스페이스)를 할 수 있다.

**Consequences (testable):**
- 전체 초기화 시 수식과 결과가 모두 비워진다.
- 백스페이스는 가장 최근 입력 문자 하나만 제거한다.

**Feature-specific NFRs:**
- 입력→화면 반영 지연이 체감되지 않을 것(즉각 반응).
- 키보드만으로 모든 핵심 동작(입력·평가·삭제·초기화) 수행 가능.

## 5. Non-Goals (Explicit)
- 그래프 그리기, 방정식/미적분 풀이, 기호 연산(CAS)을 하지 않는다.
- 사용자 계정·로그인·서버 저장을 두지 않는다 — 클라이언트 단독 동작.
- 단위 변환·진법 변환·통계 패키지를 v1에서 제공하지 않는다.
- [NON-GOAL for MVP] 공학 함수(삼각·로그·지수·팩토리얼)와 각도 모드(Deg/Rad), 상수(π·e), 메모리(M+/MR)는 v1 범위 밖 — v2 확장 후보.

## 6. MVP Scope

### 6.1 In Scope
- 기본 사칙연산(+ − × ÷), 소수점, 부호변경, 괄호.
- 버튼 + 키보드 입력.
- 수식 평가(우선순위·괄호 반영), 결과 표시.
- 오류 처리, 전체 초기화, 백스페이스.
- 단일 화면 React 웹 앱.

### 6.2 Out of Scope for MVP
- 공학 함수·각도 모드·상수·메모리 — [NOTE FOR PM] 제품명이 "공학용 계산기"이므로 v2에서 우선 추가 후보. 타임라인 허용 시 재검토.
- 계산 히스토리(이전 계산 기록) — 간결함 우선으로 v1 제외.
- 결과 복사/공유, 테마 설정, 다국어 — v1 제외.

## 7. Success Metrics
- **SM-1 (정확성):** 정의된 테스트 수식 집합에 대해 계산 결과 오류 0건. Validates FR-2.
- **SM-2 (안정성):** 잘못된 입력으로 앱이 크래시되는 사례 0건. Validates FR-3.
- [ASSUMPTION] 학습용 도구이므로 정량 사용 지표(DAU 등)는 추적하지 않는다. 성공 = "정확하고 안 멈추는 계산기".
